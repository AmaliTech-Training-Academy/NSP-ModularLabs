
## Learning Objectives
- Identify and apply appropriate prototyping triggers to match specific interaction goals  
- Demonstrate the ability to chain multiple triggers and actions to create realistic user flows  
- Implement responsive design changes that adapt layouts, navigation, and content for different screen sizes  
- Differentiate between decorative animations and functional animations that guide the user  
- Justify the choice of triggers and actions based on usability principles and device context (e.g., avoiding hover effects in mobile view)  
- Optimize interactions for both performance and accessibility  

---

## Activity Outlines

### 1. Hamburger Menu Animation
**Scenario:** In mobile view, the hamburger icon should animate into a close “X” icon when tapped.  
**Action:** Animate icon change + slide-out side navigation  

---

### 2. Hover-Activated Quick Links
**Scenario:** In desktop view, hovering over a navigation item reveals a dropdown menu with at least three sub-links.  
**Action:** Show dropdown + subtle fade-in animation  

---

### 3. Scroll-Based Sticky Header
**Scenario:** The header remains fixed after scrolling past the hero section and slightly shrinks in height.  
**Action:** Change header size + background color  

---

### 4. Animated Call-to-Action Button
**Scenario:** A CTA button expands slightly on hover and navigates to a form page when clicked.  
**Action:** Scale up → Navigate to another frame  

---

### 5. Modal Loan Calculator
**Scenario:** Clicking a function (e.g., loan calculator) in the hero section opens a centered modal with a blurred background.  
**Action:** Show overlay modal + background blur  

---

### 6. Tabbed Content Switcher
**Scenario:** In the “Banking Plans” section, clicking each tab switches content instantly without scrolling.  
**Action:** Swap content group/frame with no page reload  

---

### 7. Auto-Advancing Hero Carousel
**Scenario:** The hero section cycles through three promotional banners automatically every 5 seconds, with manual navigation arrows.  
**Action:** Auto-slide → Manual arrow navigation  

---

### 8. Form Field Focus Animation
**Scenario:** When a user taps a form field, the border color changes.  
**Action:** Change style  

---

### 9. Confirmation Toast Notification
**Scenario:** Submitting the contact form displays a toast message:  
“Thank you, we’ll get back to you soon,” which disappears after 3 seconds.  
**Action:** Show toast → Hide toast

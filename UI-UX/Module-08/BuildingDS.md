
## Learning Objectives
- Understand the purpose, structure, and documentation of a design system  
- Create and manage reusable components and styles using atomic design principles in Figma or Sketch  
- Implement and maintain a design system collaboratively with developers and brand teams  
- Demonstrate the ability to build a consistent, scalable, and developer-ready library of UI elements  

---

## Activity Outlines

### Part 1: Design System Audit & Analysis

#### Task
Choose an existing digital product (e.g., a public website or app) and perform a design system audit.

#### Deliverables
- A short report (max 3 pages) identifying inconsistencies in:
  - Typography  
  - Color usage  
  - Spacing  
  - Component behaviors  
- Annotated screenshots showing repetition or visual debt  
- Recommendations for improvement (what should be standardized?)  

#### Objective Assessed
- Understand the purpose and structure of a design system  

---

### Part 2: Component Library Creation

#### Task
Build a mini design system in Figma using **atomic design principles**.

#### Required Components

- **Atoms:**
  - Colors  
  - Typography scales  
  - Grid system  
  - Icons  

- **Molecules:**
  - Buttons  
  - Form fields  
  - Toggles  
  - Input groups  

- **Organisms:**
  - Headers  
  - Cards  
  - Modals  
  - Navigation bars  

- **Templates/Pages:**
  - A sample landing page or dashboard using your system  

#### Deliverables
- A shared Figma file with clearly labeled components and variant properties  
- A clear component naming convention and folder hierarchy  
- Demonstration of reusability (linked styles and use of auto-layout)  

#### Objective Assessed
- Create and manage reusable components and styles  

---

### Part 3: Design System Documentation

#### Task
Create a documentation page/site that explains:

- Visual foundations:
  - Colors  
  - Grids  
  - Typography  
- Component usage guidelines (with do’s and don’ts)  
- Accessibility considerations:
  - Color contrast  
  - Touch targets  

#### Deliverables
- A structured Figma documentation page or Notion-style documentation mockup  
- Include rationale behind design decisions (e.g., color choices, spacing decisions)  

#### Objective Assessed
- Document standards for consistency and collaboration

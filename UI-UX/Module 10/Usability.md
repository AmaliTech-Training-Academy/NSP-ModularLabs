
## Learning Objectives
- Define usability testing concepts and differentiate between:
  - Moderated vs. unmoderated testing  
  - Remote vs. in-person testing  
- Explain the purpose of usability testing in validating design decisions and improving user experience  
- Identify key usability metrics:
  - Task success rate  
  - Time on task  
  - Error rate  
  - User satisfaction  
- Design realistic, task-based scenarios for a prototype (e.g., mobile banking app) that reflect authentic user goals  
- Conduct usability sessions effectively (moderated or unmoderated) while minimizing facilitator bias  

---

## Activity Outline

### Prototype Usage
Use the prototype of your **travel booking application**.

---

### Usability Test Execution
- Plan and run a usability test with peers acting as participants  
- Perform at least **4 realistic tasks**, such as:
  - “Find the loan calculator”  
  - “Open a savings account”  
  - (or equivalent tasks relevant to your prototype)  

---

### Results Interpretation
Analyze user performance and behavior, for example:
- If a user takes **3 minutes to complete a 30-second task**, what does this indicate?  
- Identify potential usability issues based on:
  - Delays  
  - Confusion  
  - Errors  

---

## Deliverables

### 1. Test Plan
Include:
- Objectives  
- Target users  
- Tasks  
- Testing method (moderated or unmoderated)  

---

### 2. Recording / Observation Notes
Include:
- Issues observed during testing  
- Time on task  
- Errors made by users  

---

### 3. Analysis Report
Include:
- Prioritized list of usability issues  
- Recommendations for improvements and fixes
